Python SDN app on the POX controller. Implements routing & basic firewall over a campus‑style topology (Core + Faculty + Student Housing + IT + Data Center + Internet stubs). Handles ARP, per‑subnet forwarding, and policy enforcement.

Objectives
  • Forward IPv4 traffic across distinct LAN subnets via OpenFlow rules.
  • Handle ARP separately (flood/learn) and avoid type errors on non‑IP frames.
  • Enforce basic firewall policies (deny guest→data‑center, allow trusted PCs, etc.).
Implementation
  • Topology: s1 core; s2 Faculty, s3 Student Housing, s4 IT, s5 Data Center; Internet stubs on s1:6–8.
  • Hard‑coded subnet → port map helper (no learning) for deterministic routing.
  • ARP: detected by ethertype; flooded; cached sender IP⇄MAC to short‑circuit future flows.
  • Routing: match ip, dst → output port; installs idle‑timeout flows to offload controller.
  • Firewall: priority matches on src/dst subnets; drop rules before general forwarding.
Challenges → Solutions
  • Non‑IP frames causing exceptions → Guard & early‑return; ARP first, then extract IP.
  • Asymmetric paths → Consistent subnet mapping + mirrored return rules.
  • Rule churn → Use timeouts & coarse prefixes to minimize flow table pressure.
Results & Impact
  • End‑to‑end pings across all subnets; throughput stable under concurrent flows.
  • Blocked unauthorized guest→data‑center while preserving trusted access.
  • Clear separation of control vs. data plane with concise rule sets.
